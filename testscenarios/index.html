    Сценарии тестирования для задач MVP

    Test US.001

    Позитивный сценарий: Взаимодействие с меню ресторана

    Цель: Проверить полный цикл работы с меню - от просмотра до оформления заказа

    Шаги: 1. Открыть главную страницу ресторана 2. Проверить отображение всех элементов меню: - Названия блюд - Описания - Цены - Изображения (при наличии) 3. Изменить язык интерфейса на английский 4. Применить фильтр "Десерты" 5. Выбрать блюдо "Тирамису" 6. Проверить детальную информацию: - Полное описание - Состав - Аллергены - Кнопку "Добавить в заказ" 7. Добавить блюдо в корзину 8. Нажать "Перейти к оформлению" 9. Вернуться в главное меню

    Ожидаемый результат: - Полное меню отображается корректно на выбранном языке - Фильтр по категориям работает без ошибок - Детальная информация о блюде содержит все необходимые данные - Переход между разделами происходит плавно - Система сохраняет добавленные позиции

    Негативный сценарий 1: Просмотр недоступного блюда

    Цель: Проверить обработку запроса недоступного блюда

    Шаги: 1. Ввести в поиск "Фуа-гра" (недоступное блюдо) 2. Попытаться открыть карточку блюда 3. Проверить отображение системы

    Ожидаемый результат: - Появляется четкое сообщение: "Блюдо временно недоступно" - Кнопка добавления в корзину неактивна или отсутствует - Система предлагает аналогичные доступные варианты

    Негативный сценарий 2: Обновление меню в реальном времени

    Цель: Проверить синхронизацию данных при изменении меню

    Шаги: 1. Открыть страницу с меню 2. В админ-панели изменить: - Цену "Цезарь" с 350 → 400 руб - Статус "Тирамису" на "Недоступно" 3. Дождаться автоматического обновления (до 30 сек) 4. Проверить: - Новую цену салата - Статус десерта

    Ожидаемый результат: - Изменения отражаются без перезагрузки страницы - У "Тирамису" появляется пометка "Нет в наличии" - Цена "Цезаря" обновляется корректно - Нет ошибок в консоли или визуальных артефактов

    Test US.002

    Позитивный сценарий: Работа с категориями меню

    Цель: Проверить корректность отображения и переключения между категориями блюд

    Шаги: 1. Открыть страницу меню ресторана 2. Проверить наличие основных категорий: - Закуски - Основные блюда - Десерты - Напитки 3. Выбрать категорию "Десерты" 4. Убедиться: - В заголовке отображается название категории - Имеется краткое описание категории - В списке присутствуют только десерты 5. Переключиться на категорию "Напитки" 6. Проверить: - Мгновенное обновление списка - Соответствие блюд выбранной категории 7. Вернуться к полному списку категорий 8. Выбрать "Закуски" и проверить содержимое

    Ожидаемый результат: - Все категории отображаются с названиями и описаниями - Переключение между категориями происходит мгновенно - В каждой категории показываются только соответствующие блюда - Возможность вернуться к полному списку категорий

    Негативный сценарий 1: Выбор несуществующей категории

    Цель: Проверить обработку запроса несуществующей категории

    Шаги: 1. Вручную ввести в адресной строке "/menu/несуществующая-категория" 2. Или выбрать из фильтров "Морепродукты" (при отсутствии такой категории) 3. Проверить реакцию системы

    Ожидаемый результат: - Отображается понятное сообщение: "Категория не найдена" - Предлагаются доступные категории для перехода - Нет ошибок в консоли или визуальных артефактов

    Негативный сценарий 2: Фильтрация по недоступным типам

    Цель: Проверить поведение системы при применении нерелевантных фильтров

    Шаги: 1. Открыть страницу меню 2. В фильтрах выбрать: - "Веганские" (при отсутствии таких блюд) - Ценовой диапазон 5000-10000 руб 3. Применить фильтры 4. Проверить отображение

    Ожидаемый результат: - Показывается сообщение: "По выбранным фильтрам блюд не найдено" - Кнопка "Сбросить фильтры" активна и работает - Основное меню не искажено - Возможность выбрать другие фильтры

    Test US.003

    Позитивный сценарий: Добавление и изменение количества блюд

    Цель: Проверить корректность работы с количеством позиций при формировании заказа

    Шаги: 1. Открыть раздел "Основные блюда" 2. Выбрать "Бургер классический" (цена 450 руб) 3. Установить количество: 2 4. Нажать "Добавить в корзину" 5. Проверить: - Появление тоста "2 × Бургер добавлен в корзину" - Обновление счетчика в шапке сайта (+2) 6. Перейти в корзину 7. Проверить: - Наличие позиции "Бургер классический" - Количество: 2 - Сумму: 900 руб 8. Изменить количество на 3 9. Нажать "Обновить корзину" 10. Убедиться: - Количество изменилось на 3 - Сумма пересчиталась на 1350 руб - Итоговая сумма обновилась

    Ожидаемый результат: - Четкое визуальное подтверждение добавления - Корректное отображение и пересчет при изменении количества - Сохранение изменений при обновлении

    Негативный сценарий 1: Некорректное количество

    Цель: Проверить валидацию ввода количества

    Шаги: 1. Выбрать "Салат Цезарь" 2. В поле количества ввести: - Вариант 1: 0 - Вариант 2: -1 - Вариант 3: пустое значение 3. Нажать "Добавить в корзину"

    Ожидаемый результат: - Для 0 и отрицательных чисел: сообщение "Введите количество от 1 до 10" - Для пустого значения: восстановление значения по умолчанию (1) - Позиция не добавляется в корзину - Счетчик в шапке не изменяется

    Негативный сценарий 2: Добавление недоступного блюда

    Цель: Проверить обработку попытки добавления удаленного блюда

    Шаги: 1. Добавить "Тирамису" в корзину 2. Через админ-панель сделать блюдо недоступным 3. Обновить страницу меню 4. Попытаться повторно добавить "Тирамису" 5. Проверить состояние корзины

    Ожидаемый результат: - Сообщение "Тирамису временно недоступен" - В корзине сохраняется предыдущая добавленная позиция - Цена и доступность других блюд не изменяются - Кнопка добавления неактивна (или заменена на "Нет в наличии")

    Test US.004

    Позитивный сценарий: Успешный просмотр финального заказа с детализацией Цель: Проверить, что пользователь видит полную разбивку заказа по блюдам с корректными ценами и может перейти к оформлению.

    Шаги: 1. Добавить в корзину: - "Салат Цезарь" (2 шт) - "Гаспачо" (1 шт) 2. Перейти в корзину 3. Проверить отображение: - Названия обоих блюд - Количества (2 и 1 соответственно) - Цены за единицу - Суммы по каждой позиции 4. Проверить расчет: - Промежуточной суммы - Налогов (если есть) - Итоговой суммы 5. Нажать "Оформить заказ" 6. Убедиться в сохранении данных заказа

    Ожидаемый результат: Все блюда отображаются с правильными названиями, количеством и ценами. Все суммы рассчитаны корректно. Данные сохраняются при переходе к оформлению.

    Негативный сценарий 1: Просмотр пустой корзины Цель: Проверить отображение пустой корзины.

    Шаги: 1. Очистить корзину 2. Перейти в раздел корзины 3. Проверить: - Сообщение о пустой корзине - Отсутствие кнопки оформления - Наличие кнопки возврата в меню

    Ожидаемый результат: Отображается сообщение "Ваша корзина пуста". Кнопка оформления недоступна. Есть возможность вернуться в меню.

    Негативный сценарий 2: Изменение цен после добавления в корзину Цель: Проверить обработку изменения цен.

    Шаги: 1. Добавить "Стейк" по цене 1500 руб 2. Имитировать изменение цены до 1700 руб 3. Перейти в корзину 4. Проверить: - Отображаемую цену - Наличие уведомления - Итоговую сумму

    Ожидаемый результат: Система либо сохраняет исходную цену, либо явно уведомляет об изменении. Итоговая сумма корректна. Пользователь информирован о ситуации.

    Test US.005

    Позитивный сценарий: Успешное удаление товара из корзины Цель: Проверить возможность удаления позиции из корзины и автоматический пересчет суммы.

    Шаги: 1. Добавить в корзину: - "Салат Греческий" (1 шт) - "Суп Том Ям" (2 шт) 2. Перейти в корзину 3. Нажать иконку удаления рядом с "Суп Том Ям" 4. Подтвердить удаление во всплывающем окне 5. Проверить: - Исчезновение удаленной позиции - Обновленную общую сумму - Появление уведомления "Позиция удалена" 6. Убедиться, что оставшиеся позиции не изменились

    Ожидаемый результат: Удаленная позиция больше не отображается в корзине. Общая сумма уменьшилась на стоимость удаленного товара. Появилось подтверждение удаления. Оставшиеся позиции сохранились без изменений.

    Негативный сценарий 1: Удаление последнего товара из корзины Цель: Проверить поведение системы при опустошении корзины.

    Шаги: 1. Добавить "Чизкейк" (1 шт) 2. Перейти в корзину 3. Удалить "Чизкейк" 4. Проверить: - Сообщение "Корзина пуста" - Исчезновение кнопки "Оформить заказ" - Появление кнопки "Вернуться в меню"

    Ожидаемый результат: Отображается сообщение о пустой корзине. Кнопка оформления стала недоступной. Появилась кнопка возврата в меню.

    Негативный сценарий 2: Отмена удаления товара Цель: Проверить возможность отмены удаления позиции.

    Шаги: 1. Добавить "Ролл Филадельфия" (3 шт) 2. Перейти в корзину 3. Нажать иконку удаления 4. Во всплывающем окне нажать "Отмена" 5. Проверить: - Сохранение позиции в корзине - Отсутствие изменений в общей сумме

    Ожидаемый результат: Товар остался в корзине. Общая сумма не изменилась. Уведомление об удалении не появилось.
